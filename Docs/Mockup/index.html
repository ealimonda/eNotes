Studio di fattibilità

Il progetto assegnato richiede la realizzazione di un blocco note elettronico. Le singole note sono memorizzate usando il formato JSON, che quindi conterrà al suo interno tutti i dati della nota stessa (testo semplice e audio opportunamente codificato). Ad ogni nota sarà possibile allegare un contenuto multimediale di dimensione limitata (immagine, audio e video).

L'insieme delle note è memorizzato nel dispositivo utilizzando il database di Android (sqlite), secondo la seguente struttura: id/timestamp ultima modifica/nome della nota/json (json è memorizzato nel db).

Nel dettaglio l'applicazione dovrà avere le seguenti funzionalità:

Note List

La schermata principale è costituita da un elenco delle note organizzato in 2 colonne: nome della nota scelto dall'utente e data di ultima modifica. L'elenco delle note è “scrollabile” ed è ordinato per data di ultima modifica, la più recente in alto.

Sotto il nome di ciascuna nota, inoltre, sarà visualizzato l'elenco dei tag associati ad essa.

Questa activity è definita nella classe NoteList.

Note List (Menu)

Dalla schermata principale, tramite il pulsante menù di android, è possibile: creare una nota, vedere le note ordinate per ultima modifica (visualizzazione di default), eseguire una ricerca per tag.

View Note

Selezionando, con un click, una nota dall'elenco della schermata principale, si apre un'altra schermata che mostra il testo esteso della nota, i tag e gli allegati associati ad essa, compresi eventuali URL (cliccabili).

Questa activity è definita nella classe NoteView

View Note (Menu)

Attraverso il menu di Android di questa pagina è possibile accedere alla modalità di modifica della nota, cancellare la stessa o inviarla tramite e-mail

La nota può essere spedita tramite mail allegando il file che contiene al suo interno tutta la nota codificata in json. L'invio avviene tramite il programma predefinito nel dispositivo per l'invio di messaggi di posta elettronica.

Un utente può ricevere una nota e importarla aprendola con l'applicazione, scaricandola ad esempio come allegato da una mail.

Compose Note

In questa schermata è possibile inserire del testo semplice, scegliere un nome per la nota o associare dei tag ad essa.

Quseta activity è definita nella classe NoteEdit.

Compose Note (Menu)

Attraverso il menu della pagina è inoltre possibile:

  • inserire un link (cliccabile) ad una pagina web (un solo indirizzo per nota)
  • Allegare foto/audio/video importati da file o creati dal dispositivo (con il limite di un allegato per ogni nota)

Compose Note (URL)

L'inserimento di URL avviene tramite un'apposita finestra di dialogo.

Compose Note (Attach)

La scelta del tipo di file da allegare avviene tramite un menu, seguito dall'avvio dell'applicazione Android associata all'azione scelta oppure la finestra di ricerca di file nella galleria, dove opportuno.

View Note (Delete)

Le operazioni di eliminazione di una nota avverranno previa richiesta di conferma da parte dell'utente, tramite un'apposita finestra di dialogo.

Search

L'utente può associare una o più etichette o tag ad ogni nota.

Questi tag permettono la ricerca delle note tramite parola chiave. La ricerca avviene inserendo in una casella di testo un determinato tag oppure selezionandone uno da una lista di tag esistenti. Tale lista filtra in automatico i tag durante la digitazione.

Questa activity è definita nella classe NoteSearch.

Analisi dei requisiti:

Per il funzionamento dell'applicazione sono necessari i seguenti requisiti:

  • fotocamera abilitata per l'acquisizione di foto/video
  • microfono abilitato per l'acquisizione dell'audio
  • gestione del formato JSON che conterrà i dati della nota
  • memorizzazione delle note nel database di android sqlite

Definizione di un'architettura logica:

Le classi eNotes, editNotes, searchNotes e composeNotes corrispondono ciascuna ad una delle activity presenti nell'applicazione. La classe Note modella la singola nota mentre la classe dbNotes agisce da backend per il database.

  • NoteList: Definisce l'activity principale dell'applicazione, e viene mostrata quando l'applicazione viene avviata. In questa activity viene mostrata l'interfaccia principale dell'applicazione (l'elenco delle note) e vengono creati e gestiti i vari menù per le diverse funzionalità offerte come: creare, cercare, visualizzare una nota per data di ultima modifica. Tramite questa interfaccia l'utente potrà scorrere le note e selezionarle.
  • NoteView: activity chiamata da quella principale, che consente la visualizzazione di una nota per esteso mostrandone il testo contenuto. Il menu incluso permette di modificare, cancellare o inviare una nota tramite e-mail.
  • NoteSearch: activity chiamata da quella principale che consente la ricerca di una nota tra quelle memorizzate nel database. Permette, inoltre, di filtrare i tag esistenti durante la digitazione da parte dell'utente nel campo di ricerca.
  • NoteEdit: activity chiamata da NoteView (modalità di modifica) o da NoteList (modalità di creazione), gestisce la composizione di una nota e l'aggiunta dei tag per la sua ricerca. Permette, inoltre, la possibilità di inserire (attach) in una nota, un contenuto multimediale (immagine, video, audio) ottenuto tramite operazione di import/capture. Quando invocata in modalità di modifica, i campi sono pre-compilati con i contenuti della nota esistenta, altrimenti sono vuoti.
  • Note: rappresenta la singola nota e ne contiene tutti i dati, gestendo anche la trasformazione da/a JSON. Per ogni nota visualizzata o modificata, verrà istanziato un oggetto di questo tipo.
  • NoteDB: agisce da interfaccia con il database sqlite per la memorizzazione delle note. Gestisce quindi tutte le possibili operazioni consentite come il recupero, la modifica, la cancellazione e la ricerca di una nota.
